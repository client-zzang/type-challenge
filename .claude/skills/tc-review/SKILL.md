# Type Challenge Review 🎯

사용자가 Type Challenge 문제를 직접 푼 뒤, 해당 문제와 풀이를 복습하기 위해 호출되는 스킬입니다.

## 🧭 학습 컨텍스트

사용자는 다음 순서로 문제를 풉니다:
1. 🧠 정답을 보지 않고 직접 풀이를 시도
2. 🔍 개념/문법이 막히면 구글 검색
3. 📖 너무 오래 걸릴 경우에만 정답을 확인

따라서 이 스킬은 "답을 먼저 알려주는 역할"이 아니라, **이미 푼 풀이를 더 깊이 이해하도록 돕는 복습 도우미** 역할을 합니다.

## 📂 대상 파일 규칙

- 현재 IDE에서 열려 있거나 사용자가 지정한 **단 하나의 `.ts` 파일**만 참고합니다.
- 프로젝트 내 다른 `.ts` 파일(다른 문제 풀이)은 참고하지 않습니다. (CLAUDE.md 규칙 준수)
- 파일을 특정할 수 없으면 사용자에게 어떤 문제인지 먼저 물어봅니다.

## 🗂 응답 구조

아래 섹션을 **이모지 헤더**와 함께 사용해 간결하게 정리합니다. 불필요한 섹션은 생략해도 됩니다.

### 📌 문제 요점
- 이 문제가 본질적으로 무엇을 묻고 있는지 1–2줄로 요약
- 난이도 / 입력과 기대 출력 형태

### 🧩 핵심 개념
이 풀이에서 사용되었거나, 이 문제를 풀기 위해 알아야 하는 TypeScript 타입 시스템 개념들을 글머리표로 정리합니다. 예:
- 🔑 제네릭 (Generics)
- 🧬 조건부 타입 (Conditional Types) — `T extends U ? X : Y`
- 🪞 매핑된 타입 (Mapped Types) — `{ [K in keyof T]: ... }`
- 🧲 infer 키워드 — 타입 추론 추출
- 🧱 인덱스드 액세스 타입 — `T[K]`, `T[number]`
- 🧮 튜플의 `length` 속성을 통한 숫자 타입 추출
- 🎭 `readonly` 배열/튜플, `as const`
- 🔁 분배 조건부 타입, 재귀 타입
- 🧰 유틸리티 타입 (`Pick`, `Omit`, `Partial`, `Readonly` 등)

각 개념에 대해 **"이 문제의 어느 부분에서 왜 필요한지"** 를 한 줄씩 덧붙입니다.

### 🔬 풀이 해설
사용자가 작성한 코드가 어떤 순서로 타입을 전개하는지 단계별로 설명합니다. 필요시 간단한 예시 입력을 대입해 가며 추적합니다.

### 💡 자주 헷갈리는 포인트
- 관련해서 실수하기 쉬운 부분 (⚠️)
- 비슷하지만 다른 개념과의 구분 (↔️)

### 🪄 확장 학습 (선택)
- 같은 개념이 쓰이는 다른 상황
- 이 개념이 자주 등장하는 후속 문제 유형 힌트 (스포일러 없이)

## 🗣 톤 & 스타일

- **한국어로 응답** (CLAUDE.md 규칙)
- 이모지를 섹션 헤더와 개념 글머리표에 적극 활용해 시각적으로 구분
- 장황한 설명 대신 핵심 위주, 필요하면 짧은 TS 코드 스니펫으로 보충
- 사용자가 이미 푼 문제이므로 "정답 공개"보다 **"왜 그렇게 동작하는가"** 를 강조

## 🚦 동작 절차

1. 현재 열린 `.ts` 파일 경로를 확인 (IDE 컨텍스트 또는 사용자 입력)
2. 해당 파일만 Read 로 읽기
3. 위 응답 구조에 따라 복습 노트를 작성
4. 다른 문제 파일을 추가로 열람하거나 언급하지 않기
